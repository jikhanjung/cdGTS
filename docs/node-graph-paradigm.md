# 노드 그래프 패러다임 (The Node-Graph Paradigm)

> cdGTS의 핵심 개념적 은유. 블렌더의 지오메트리 노드(Geometry Nodes)에서 출발.

## 은유

블렌더 지오메트리 노드처럼, cdGTS는 **노드들의 네트워크**로 구성된다:

- **데이터 노드** — 다양한 원시 관측 (방사연대, 천문연대, 자기층서, 생층서 …). 불변(immutable), 인용 가능(cited), 그래프의 리프(leaf).
- **프로세스/모델 노드** — 입력을 변환하는 처리·모델 (보정, age-depth 모델, correlation …).
- **엣지** — 데이터 흐름.

이 네트워크가 **평가(evaluate)** 되면 결과적으로 **지구 위 특정 지점에 대한 연대**가 결정된다. 이 모든 것을 모으면 ICC / GTS2030 같은 것이 된다.

이 관점은 앞서 [idea.md](idea.md)의 Layer 2(데이터) → Layer 3(모델) → Layer 4(출력) 구조와 겹치되, "계층"보다 **DAG(방향성 비순환 그래프)** 로 보는 것이 더 강력하다.

## 노드 메타포가 자동으로 주는 것

- **Provenance = 그래프 역추적.** 어떤 경계 연대에서 엣지를 거꾸로 따라가면 기여한 모든 소스가 드러난다. FAIR 원칙의 lineage/audit가 별도 기능이 아니라 **구조 자체에 내장**된다.
- **증분 재평가 (Incremental re-evaluation).** 바뀐 노드의 하류만 다시 계산 → "continuously tested"의 엔진.
- **노드 스왑 = what-if.** 모델 노드를 스플라인 ↔ 베이지안으로 교체하거나 데이터 노드를 갈아끼우면 출력이 어떻게 달라지는지 diff. 대안 모델·데이터 = 대안 그래프 브랜치.
- **Node group = 지역/경계별 서브그래프.** 특정 노두(locality)의 로컬 그래프들이 correlation 노드를 통해 글로벌 스테이지 경계에 묶이고, 이를 모두 합치면 전 지구 시대표. → "지구 위 특정 지점"이라는 표현대로 진짜 **공간(geometry) 차원**이 존재.

## 두 산출물, 하나의 그래프

같은 그래프에서 두 가지가 나온다:

- **GTS2030** = 그래프를 **길게 서술(narrate)** 한 것 — 추론·근거·caveat까지 직렬화한 "책".
- **ICC** = 그래프를 **bake / crystallize** 한 스냅샷 — 최종 경계 숫자 + 이름만 얼린 것. (블렌더의 "modifier 적용 → 메시 굽기"에 대응.)

## 블렌더와 다른 지점 — 어려운 부분

1. **엣지가 스칼라가 아니라 분포(distribution)를 흘린다.** 각 관측엔 불확실성이 있으므로 평가가 결정론적이지 않고 **확률적 전파**(몬테카를로 / 베이지안)여야 한다. "bake"란 곧 분포를 요약·고정하는 행위.
2. **순환 의존성.** 생층서는 방사연대로 보정되고, 그 방사연대 시료의 위치는 다시 생층서로 잡힌다 — 순수 DAG가 깨질 수 있다. 순환을 금지하고 명시적 **공동추정(joint inference) 노드**로 접을지, 반복 수렴을 허용할지 결정 필요.

## 추가로 남는 문제

- **토폴로지도 버전 대상.** 노드 값뿐 아니라 배선(wiring) 자체가 바뀐다.
- **거버넌스 / 큐레이션.** "어느 그래프가 *공식* ICC 그래프인가"를 누가 어떻게 정하는가.

## 게이트웨이 레이어 — 계약과 네트워크

> 상태: hunch. 실제 사례로 검증이 필요하다. [idea.md](idea.md) §8과 짝을 이룬다.

레이어([idea.md](idea.md)의 Layer 0~4)를 "파이프라인 단계"가 아니라 **계약(contract)** 으로 재해석한다. 두 종류의 것이 번갈아 나온다:

- **게이트웨이 레이어** = 모두가 합의한 **고정 타입/인터페이스**. 인용 가능, 버전 태깅, 릴리스의 단위. "여기서만큼은 이 스키마로 얼린다"는 **커밋 지점**.
- **게이트웨이 사이의 노드 네트워크** = 그 타입을 만들어낸 **자유로운 위상**. 순환·대안 모델·확률 전파·correlation이 사는 곳.

핵심 규칙: **네트워크는 자유롭되 게이트웨이 타입은 지킨다.** 스플라인을 쓰든 베이지안을 쓰든, 둘 다 같은 게이트웨이 타입(예: "글로벌 경계 추정 = 분포 + provenance")을 뱉어야 한다. → 경쟁 모델 = what-if 브랜치.

| | 게이트웨이 | 노드 |
|---|---|---|
| 위상 | 고정, 스키마 계약 | 자유, 교체 가능 |
| 정체성 | 인용·주소지정 가능 | 익명 가능 |
| 버전 | 독립 태깅 | 네트워크째로 흐름 |
| 불변성 | 릴리스되면 얼림 | 계속 churn |
| 거버넌스 | **비준(ratify) 대상** | 자유 실험 |

이게 주는 것:

- **거버넌스 축소.** "모든 노드를 통제"가 아니라 **"어느 게이트웨이를 비준하느냐"**. 학자는 게이트웨이 사이 네트워크에서 자유롭게 놀고, 큐레이터(ICS 등)는 게이트웨이만 도장 찍는다.
- **순환 격리.** 순환은 두 게이트웨이 *사이* 영역에 갇혀 joint inference로 풀리고, 밖으로 나오는 게이트웨이 출력은 깔끔한 분포. 순환이 전 그래프로 번지지 않는다.
- **CD의 의미가 또렷해진다.** 게이트웨이는 주기적으로 얼리고(릴리스), 사이 네트워크는 계속 돈다. CI diff = "이 네트워크를 건드리면 다음 게이트웨이 숫자가 어떻게 바뀌나".

비유: 컴파일러의 **IR 체크포인트.** 게이트웨이 = 직렬화·검사·diff 가능한 안정된 중간표현(IR), 사이 네트워크 = 언제든 갈아끼울 수 있는 최적화 패스.

대략적 배치 (idea.md의 Layer를 게이트웨이로 읽으면):

```
[G:명명]─▷(net)─▷[G:원시관측]─▷(age-depth net)─▷[G:국소 보정 section]
                                                        │
                                        (correlation + synthesis net)
                                                        ▼
                                              [G:글로벌 경계 추정]
                                                        │
                                              (bake / narrate net)
                                                  ┌─────┴─────┐
                                                  ▼           ▼
                                              [G:ICC]     [G:GTS]
```

열린 질문:

- 게이트웨이는 **전역**인가 **경계별**인가.
- **타입(스키마)** 인가 **얼린 인스턴스(릴리스)** 인가, 둘 다인가.
- 무엇을 게이트웨이로 **승격**하는가 = 거버넌스의 경계.
- **경계 게이트웨이는 다형(polymorphic).** GSSP형(계산된 분포, ± 있음) ↔ GSSA형(결정된 상수, 오차 없음)
  두 타입을 한 스키마로 담아야 한다. 근거: [case-permian-triassic.md](case-permian-triassic.md),
  [case-precambrian-gssa.md](case-precambrian-gssa.md).
- **토폴로지 재배선도 diff 대상.** GSSA leaf → GSSP 네트워크 전환처럼 배선 자체가 바뀌면
  값 diff와 별개로 **위상 diff**가 필요하다 (Ediacaran 완료, Cryogenian 진행 중 — 실제 진행형 사례).
