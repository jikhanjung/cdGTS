# 20260705_057 — 노드그룹 선택·병합 + order 포트

> "그룹도 노드처럼 선택해서 그룹+노드를 하나로 묶고 싶은데 선택이 안 된다" + "그룹도 order 위아래 포트 필요".

## 한 일 (`Editor.jsx`)
- **그룹 선택**: `onSelectionChange` 가 접힌 그룹을 `selectedGroupKeys` 로 수집(실노드는 `selectedIds`).
  `buildView` 가 그룹 뷰노드에 `selected` 를 주입(합성 노드라 선택이 리셋되던 문제 해결).
- **병합**: `createOrMergeGroup(realIds, groupKeys)` — 선택에 그룹이 섞이면 그 멤버까지 흡수해 **단일 계층 병합**
  (대상=첫 그룹, 나머지 선택 그룹은 합쳐지며 소멸). 순수 노드면 새 그룹. 툴바 "그룹 만들기/병합"·"그룹 해제(N)",
  우클릭 메뉴(노드/그룹/빈곳)도 병합 지원.
- **그룹 order 포트**: 접힌 그룹은 경계 넘는 older/younger 엣지를 위/아래 포트로 이미 표시([369d8eb]).
  추가로 `onConnect` 가 **그룹 포트↔멤버 포트 환원**(`out:<member>:<port>`/`in:...`) → 그룹의 세로 포트에서
  order 를 새로 그릴 수 있음.

## 한계 (이월)
- 단일 계층만 — 그룹 안 그룹(중첩)은 미지원. "병합"은 평탄화.
- 기존 crossing 엣지가 있는 포트에서만 그리기 가능(자유 order 포트는 멤버 모호성으로 보류).

## 검증
- 프론트 빌드 클린. 백엔드·시드·마이그레이션 무관. **브라우저 육안 필요**(그룹 선택 하이라이트·병합·그룹 포트 연결).
