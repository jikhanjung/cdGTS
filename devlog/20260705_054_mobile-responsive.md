# 20260705_054 — 폰 대응 (반응형 · 터치)

> "폰에서도 원활히 작업 가능한지" — 뷰 화면은 볼 만하나 **에디터가 3단 고정폭**(팔레트 220 | 캔버스 | 인스펙터 280)
> 이라 폰에서 깨지고, `panOnDrag=[1]`(가운데버튼)이라 **터치 팬 불가**. 반응형으로 수정.

## 진단
- 데스크톱 전용 3단 레이아웃 → ≈375px 에서 캔버스 음수폭.
- topnav 5버튼·toolbar 가로 오버플로. `.icctable`(min 560)·`.difftable`(min 480) 가 컨테이너 밖으로 넘쳐 바디 가로 스크롤.
- 노드 정밀 배선은 터치에서 본질적 한계 — **보기 + 가벼운 편집(팬/줌·탭 선택·뷰 전환)** 을 목표로.

## 한 일
- **터치 상호작용** (`Editor.jsx`): `matchMedia('(pointer: coarse)')` 로 감지 →
  터치는 `panOnDrag=true`(드래그 팬)+`zoomOnPinch`, 데스크톱은 기존(좌-드래그 선택·가운데 팬).
- **팔레트/인스펙터 서랍** (≤820px): 캔버스 풀폭, 팔레트는 좌·인스펙터는 우에서 슬라이드. 툴바에 `☰`(팔레트)·`속성`
  토글(모바일 전용), backdrop 탭으로 닫기. 인스펙터에 모바일 닫기 `✕`.
- **CSS 미디어쿼리** (≤820px): topnav 가로 스크롤, toolbar wrap·status 개행, 뷰 여백 축소.
  `.view{overflow:hidden}` + `.icc/.diff{overflow-x:auto}` 로 넓은 표는 컨테이너 안에서만 스크롤(바디 가로 스크롤 차단).

## 검증
- 프론트 빌드 클린. 백엔드·시드 무관. **실제 폰 육안 확인 필요**(헤드리스 브라우저 없음): 서랍 슬라이드·터치 팬/핀치·표 스크롤.

## 남는 한계
- 노드 생성은 팔레트 **드래그&드롭** — 터치 DnD 는 불안정할 수 있음(후속: 탭-투-추가 고려).
- 우클릭 컨텍스트 메뉴(그룹화)는 터치에서 롱프레스 매핑 필요(후속).
