# 20260704_039 — 노드그룹 (컨테이너 + 드릴인)

> 휴면 스텁이던 `graph.NodeGroup` 을 실기능으로. 그룹을 만들어 안에서 노드를 편집하고, 접으면
> 경계 넘는 엣지가 자동으로 입출력이 되는 컨테이너. **결정: 컨테이너+접기(엔진 평탄) · 드릴인 편집.**

## 결정 (사용자 선택)

- **그룹 성격 = 컨테이너+접기** — 그룹은 멤버십 + 접기 + 위치. 데이터흐름은 평탄(엣지는 노드↔노드 그대로),
  **엔진 무변경**. 접으면 경계 넘는 엣지가 그룹 노드의 입출력 핸들로 자동 정합.
- **내부 편집 = 드릴인** — 그룹 더블클릭 → 그룹 멤버만 보이는 하위 캔버스, breadcrumb 로 복귀.

## 한 일

### 백엔드 (왕복만, 엔진 무관)
- `NodeGroup` 에 `x/y`(접힌 노드 위치) 추가 — migration `graph.0003`.
- `GraphSerializer`: `groups[]`(key·name·collapsed·x·y) + `NodeInstance.group`(소속 그룹 key, write/read)
  왕복. `_replace_topology` 가 그룹 재생성 + 멤버 배정. 검증: 노드의 group key 는 groups 에 있어야.
- 엔진/evaluate/bake 는 그대로 — 그룹은 표현·편집용이라 전체 노드 평탄 평가.

### 프론트 (`Editor.jsx` + `GroupNode.jsx`)
- 상태 = **전체 실제 노드/엣지** + 그룹 메타 + `activeGroup`(null=최상위 / key=드릴인). ReactFlow 에 넘길
  `{viewNodes, viewEdges}` 는 `buildView` 로 파생.
- **최상위**: 그룹 = 접힌 노드(`cdgtsGroup`). 멤버는 숨김, 경계 넘는 엣지는 그룹 노드 입출력 핸들로 재라우팅
  → 입출력 자동 정합. 그룹 내부 엣지는 숨김.
- **드릴인**: 멤버 + 내부 엣지 편집. 경계 넘는 연결은 좌(입력)/우(출력) **인터페이스 스텁**(`cdgtsStub`,
  읽기전용)으로 표시.
- **조작**: 노드 선택 → "그룹 만들기"(centroid 위치), 그룹 선택/드릴인 → "그룹 해제", 더블클릭 드릴인,
  breadcrumb 복귀. 드릴인 중 팔레트 드롭 = 그 그룹에 멤버 추가.
- 변경 매핑: `onNodesChange` 실제 노드→상태 / 그룹 노드→위치 / 스텁→무시. `onConnect` 는 실제 노드끼리만.
  저장(`rfToApi`)은 전체 실제 노드+엣지+그룹 메타 왕복.
