### 버그 수정 — `seed --mode=replace` ProtectedError (커밋 62b9356)
- **증상**: 운영(데이터 있는 DB)에서 replace 시 `chrono.Unit` 삭제가 **ProtectedError** 로 실패.
  `Unit.parent` 가 자기참조 **PROTECT** FK라, 자식이 부모를 참조하는 상태에서 일괄 delete 가 막힘.
- **수정**: `_delete_all()` 이 삭제 전에 SEED_MODELS 의 **nullable self-FK 를 일괄 null 로 끊는다**
  (`Unit.parent`, `Authority.parent`). 그 후 역-의존 순서 delete 가 통과.
- **놓친 이유(테스트 갭)**: 로컬 검증은 replace 를 *빈/신선 DB*(deleted 0)에서만 돌려 자기참조 삭제 경로를
  건드리지 않았다. 데이터가 있는 DB 에서의 replace 를 테스트했어야 함.
  → 로컬 재현: 시드된 DB 에 replace 재실행 시 동일 오류, 수정 후 통과 확인.
