# 20260705_073 — 릴리스 0.1.17 (order 노드 유령 폭 = 선택 판정 폭 버그 픽스)

> [072](20260705_072_release-0.1.16.md)(0.1.16, 선택 Full 모드) 이후 **진짜 근본 원인**을 잡음. 배포는 사용자.

## 증상 (사용자 정밀 진단)
order 노드(화면상 40px `≥` 글리프)를 선택하려고 그 위를 감싸도 선택이 안 되고, **data/process 노드 폭만큼**
오른쪽까지 selection box 를 넓혀야 그제서야 선택됨. 즉 React Flow 가 order 노드를 실제 40px 가 아니라
**data 노드 폭으로 판정**하고 있었다("몇 개의 고정 x2 값" = 그 유령 오른쪽 끝).

## 원인
`apiToRF`/`addNodeAt` 가 **모든** 노드에 명시적 `width: n.width || DEFAULT_NODE_WIDTH`(=172) 를 넣었다.
React Flow 는 명시 width 가 있으면 측정 대신 그 값으로 bounds·hit-test 를 계산한다. order 노드는
CSS(`.order-node`)로 40px 로 보이지만 판정 폭은 172px → 오른쪽으로 132px 유령 영역.
(data/process 는 `.cdgts-node{width:100%}` 라 래퍼 폭이 필요해 명시 width 가 맞다.)

## 고침 (`Editor.jsx`)
- `nodeWidth(slug, w)` — order 타입이면 `undefined`(→ React Flow 실측 40px), 그 외는 `w || 172`.
- `apiToRF`(로드)·`addNodeAt`(생성) 둘 다 적용. DB 에 이미 172 가 저장돼 있어도 order 면 무시하고 실측 사용.
- 072 의 `SelectionMode.Full` 과 합쳐, 이제 화면에 보이는 대로 order 노드를 감싸면 선택된다.
