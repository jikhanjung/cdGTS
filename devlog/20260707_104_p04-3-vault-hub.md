# 20260707_104 — P04.3: Vault 허브 (프론트)

[P04](20260707_P04_editor-bake-vault-restructure.md) 3단계. Editor 외 4개 뷰(ICC Table/Chart/Narrative,
Releases Diff)를 하나의 **Vault** 허브로 접고, 상단 nav를 **Editor · Vault** 2개로 축소.

## 변경

- **Vault.jsx (신규)** — 아티팩트 허브. 좌측 rail = Release 목록(version·kind 배지·source_graph·record_count·
  날짜, 클릭 선택). 상단 토글 = **Chart / Table / Narrative / Diff**. 선택 Release를 각 표현으로 렌더.
  `initialReleaseId`로 방금 구운 Release 자동 선택.
- **표현 컴포넌트 = release 중심으로 임베드 가능**하게 리팩터(`embedReleaseId` prop, 있으면 자체 피커 숨김):
  - `IccChart` — 임베드 시 release 모드로 해당 Release 차트, source/graph 피커 숨김(scale·zoom 유지).
  - `IccTable` — 임베드 시 `getRelease(id).records`를 테이블로(그래프 bake 대신).
  - `Narrate` — 임베드 시 해당 Release narrate.
  - `ReleasesDiff` — `initialA`로 A 프리셋. **light 목록 회귀 수정**: `records.length`→`record_count`
    (P04.1에서 목록 직렬화가 경량화돼 `records` 미포함).
- **App.jsx** — nav = Editor·Vault. `Editor onBaked={goVault}` → bake 성공 시 그 Release로 **Vault 진입**.
- **index.css** — `.vault` 레이아웃(rail + main + 모드 토글 + kind 배지), 모바일 스택.

## 검증

`npm run build` 통과(203 modules). listReleases 경량 페이로드의 `.records` 잔존 참조 없음(getRelease/bake
응답만 records 보유).

1급이 됐고, 이게 아크 C(P05)의 소유·fork·propose가 얹힐 토대. 배포는 스키마 migration 포함 재시드 필요.
