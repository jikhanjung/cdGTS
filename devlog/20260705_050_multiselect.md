# 20260705_050 — 다중 선택 조작 (좌-드래그 선택 / 중앙 팬)

> "노드 여러 개 선택이 안 되는 것 같다" — 사실 동작은 있었으나 React Flow 기본값(Shift+드래그)에 가려짐.
> 좌-드래그를 팬이 차지해 발견성 0. node editor 관례로 재배치.

## 한 일 (`Editor.jsx` ReactFlow props)
- `selectionOnDrag` — 좌-드래그 = 고무줄 선택 박스.
- `panOnDrag={[1]}` — 팬 = **가운데버튼 드래그**.
- `multiSelectionKeyCode="Shift"` — Shift+클릭 = 추가 선택.
- `selectionMode=Partial` — 박스에 걸치기만 해도 선택.
- 우클릭(컨텍스트 메뉴)·스크롤 줌·Controls 는 그대로. 선택→우클릭→그룹 흐름이 발견 가능해짐.

## 참고
- 로직(selectable·onNodesChange select 반영·onSelectionChange→selectedIds)은 이미 정상이었음 — 조작 배치만 문제.
- 트랙패드(가운데버튼 없음)는 팬이 Controls/줌에 의존 — 필요 시 `panOnScroll` 추가 고려.
- 프론트 빌드 클린. 백엔드·마이그레이션 무관.
