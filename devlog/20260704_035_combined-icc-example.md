# 20260704_035 — 조립 예제 그래프 (ICC 일부)

> 단일-경계 예제 3종을 한 네트워크로 합쳐 ICC 일부를 덮는 조립본 추가.
> seed 통합([032](20260704_032_seed-consolidation.md)) · 예제 그래프([029](20260703_029_example-graphs.md)) 후속.

## 배경

기존 예제 3개는 각각 경계 하나·유형 하나만 보여준다(GSSA 결정 / age-depth 보간 / 섹션 간 상관).
"ICC 일부를 커버하는 결과물"을 원해 → 셋을 **한 그래프**로 합쳐 여러 경계를 동시에 산출.

## 한 일

### 새 예제 `example-icc-partial`
- 세 서브그래프를 union: **9 노드 · 6 엣지 · 게이트웨이 3**. 노드 키 충돌 없어 그대로 병합.
- 각 노드 params(실제 값)는 복사, 세 서브그래프를 **세로 밴드**로 분리 배치(y 오프셋).
- 터미널 노드에 `description` 부여(제목 툴팁 데모, [034](20260704_034_node-width-description.md)).
- **기존 3개 예제는 유지**(각 경계 유형 학습용). → seed 그래프 3 → 4.

### 산출 (한 번의 평가로)
| 경계 | 연대 | 방식 |
|---|---|---|
| base-proterozoic | 2500 Ma (exact) | GSSA 결정(pin) |
| base-cambrian | 538.8 ± 0.54 Ma [decomposed] | 섹션 간 상관 |
| base-triassic | 251.9 ± 0.024 Ma [decomposed] | age-depth 보간 |

### 정합성 게이트 (단일 그래프로는 못 하던 것)
- 세 경계가 층서순으로 단조(2500 > 538.8 > 251.9) → **인증서 pass**(L1).
- 단, `engine._certify` 의 L1 은 아직 **게이트웨이 나열 순서** 기준 스텁(층서순 대조 미구현)이라,
  게이트웨이를 오래된→젊은 순으로 배치해야 통과. 실제 차트 나열 방식과 일치시켜 정렬함.
  → 후속: `_certify` 가 경계 below/above(층서)로 정렬 후 단조성 검사하도록 개선.

## 검증
- seed replace 로드(inserted 115), 새 그래프 평가 → 3경계 값 정확 + 정합성 pass 확인.
- `pytest` **63 passed**(seed 테스트 그래프 카운트 3→4 갱신).

## 반영
- seed 데이터 → 운영 반영은 다음 이미지(0.1.4, 034 마이그레이션 포함) 배포 후
  **`seed --mode=add`** 1회: 기존 데이터 보존하며 새 그래프만 원자적으로 추가(add 모드 첫 실사용).
